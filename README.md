# Nginx Proxy Chain + X-Forwarded-For

Учебный стенд: цепочка из трёх reverse proxy на nginx перед Python-приложением. Демонстрирует корректное формирование заголовка `X-Forwarded-For` при прохождении запроса через несколько прокси и защиту от подделки этого заголовка клиентом.

## Задача стенда

Когда запрос проходит через несколько прокси, приложение должно видеть полную цепочку IP-адресов: реальный IP клиента и адреса всех промежуточных прокси. При этом клиент не должен иметь возможности «подсунуть» произвольный IP в начало цепочки — заголовок `X-Forwarded-For` принимается только от доверенных прокси.

## Схема потока

Прямое обращение (любой из трёх nginx):

```
client → nginxN → app
```

Цепочка (endpoint `/chain` на любом nginx):

```
client → nginx1 → nginx2 → nginx3 → app
```

- `nginx1` — `172.30.0.11` (порт `8081` наружу)
- `nginx2` — `172.30.0.12` (порт `8082` наружу)
- `nginx3` — `172.30.0.13` (порт `8083` наружу)
- `app` — `172.30.0.20` (порт `8084` наружу), простой HTTP-сервер на Python, который возвращает полученный заголовок `X-Forwarded-For`

## Защита от подделки X-Forwarded-For

Клиент может отправить запрос с заголовком `X-Forwarded-For: 1.2.3.4`, чтобы скрыть свой настоящий адрес. На каждом nginx это отсекается через директиву `map $remote_addr`:

- если запрос пришёл **от доверенного адреса** (одного из прокси или приложения подсети `172.30.0.0/24`) — берём присланный им `X-Forwarded-For` как основу;
- если запрос пришёл **от кого угодно другого** (напрямую от клиента) — присланный заголовок отбрасывается, а основой становится реальный IP клиента (`$remote_addr`).

Далее вторая `map` дописывает к основе адрес текущего прокси. В результате поддельный заголовок удаляется на первом же nginx, а приложение всегда получает корректную цепочку, начинающуюся с реального IP клиента, например:

```
X-Forwarded-For: 172.30.0.1, 172.30.0.11, 172.30.0.12, 172.30.0.13
```

где `172.30.0.1` — адрес Docker-шлюза, то есть реальный адрес клиента.

## Стек

- Docker Compose
- nginx (alpine) — reverse proxy
- Python 3 (alpine) — тестовое приложение на стандартной библиотеке (`http.server`)

## Структура проекта

```
├── app/
│   └── app.py              # приложение, возвращающее X-Forwarded-For
├── nginx/
│   ├── nginx1.conf         # конфиг первого прокси (порт 8081)
│   ├── nginx2.conf         # конфиг второго прокси (порт 8082)
│   └── nginx3.conf         # конфиг третьего прокси (порт 8083)
├── docker-compose.yml      # оркестрация: 4 контейнера, сеть 172.30.0.0/24
└── Протокол.md             # протокол тестирования с примерами curl
```

## Быстрый старт

```bash
docker compose up --build
```

Проверить работу:

```bash
curl http://localhost:8081/        # через один прокси
curl http://localhost:8081/chain   # через всю цепочку
curl -H "X-Forwarded-For: 1.2.3.4" http://localhost:8081/chain  # попытка подделки
```

Остановить стенд:

```bash
docker compose down
```
