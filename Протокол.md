# Протокол тестирования

## Подготовка
```bash
docker-compose up -d
```

## 1. Прямое обращение к приложению
```bash
curl http://localhost:8084/
```
**Ожидаемый результат:** `X-Forwarded-For: NOT PRESENT`

## 2. Обращение через один nginx (любой из трёх)
```bash
curl http://localhost:8081/
curl http://localhost:8082/
curl http://localhost:8083/
```
**Ожидаемый результат:**
- `8081/` → `X-Forwarded-For: 172.30.0.1, 172.30.0.11`
- `8082/` → `X-Forwarded-For: 172.30.0.1, 172.30.0.12`
- `8083/` → `X-Forwarded-For: 172.30.0.1, 172.30.0.13`

*(172.30.0.1 — IP Docker-шлюза, т.е. реальный IP пользователя)*

## 3. Обращение через цепочку nginx
```bash
curl http://localhost:8081/chain
curl http://localhost:8082/chain
curl http://localhost:8083/chain
```
**Ожидаемый результат:**
- `8081/chain` → `X-Forwarded-For: 172.30.0.1, 172.30.0.11, 172.30.0.12, 172.30.0.13`
- `8082/chain` → `X-Forwarded-For: 172.30.0.1, 172.30.0.12, 172.30.0.13`
- `8083/chain` → `X-Forwarded-For: 172.30.0.1, 172.30.0.13`

## 4. Проверка защиты от поддельного заголовка (прямое обращение)
```bash
curl -H "X-Forwarded-For: 1.2.3.4" http://localhost:8081/
curl -H "X-Forwarded-For: 1.2.3.4" http://localhost:8082/
curl -H "X-Forwarded-For: 1.2.3.4" http://localhost:8083/
```
**Ожидаемый результат:**
- `8081/` → `X-Forwarded-For: 172.30.0.1, 172.30.0.11`
- `8082/` → `X-Forwarded-For: 172.30.0.1, 172.30.0.12`
- `8083/` → `X-Forwarded-For: 172.30.0.1, 172.30.0.13`

*(поддельный заголовок отброшен, использован реальный IP пользователя)*

## 5. Проверка защиты при прохождении цепочки
```bash
curl -H "X-Forwarded-For: 1.2.3.4" http://localhost:8081/chain
```
**Ожидаемый результат:** `X-Forwarded-For: 172.30.0.1, 172.30.0.11, 172.30.0.12, 172.30.0.13` — поддельный заголовок отброшен на первом nginx, далее формируется корректная цепочка.

## Завершение
```bash
docker-compose down
```
